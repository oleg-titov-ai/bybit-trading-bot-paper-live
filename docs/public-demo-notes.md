# Public Demo Notes

Use this note when preparing a public demo of the repository.

## Demo Goals

- Show paper mode first.
- Explain the risk notes clearly.
- Keep setup steps easy to repeat.
- Use demo values only.
- Keep live mode separate from the main demo.

## Before Sharing

- Check that the README is current.
- Check that examples do not show account data.
- Check that screenshots are safe.
- Check that the risk disclaimer is visible.
